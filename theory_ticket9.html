Теоретический вопрос: Объясните работу адаптера в RecyclerView и ViewPager

1. Понятие адаптера в Android

Адаптер (Adapter) — это реализация паттерна проектирования «Адаптер» в Android. Он выступает посредником между источником данных (список, массив, БД) и визуальным компонентом (RecyclerView, ViewPager), который эти данные отображает.

Адаптер решает три задачи:

  1. Сообщить виджету сколько элементов нужно показать.
  2. Создать визуальный контейнер для каждого элемента (ViewHolder / Fragment).
  3. Заполнить контейнер данными для конкретной позиции.

2. Адаптер в RecyclerView

2.1 Класс RecyclerView.Adapter

Для работы с RecyclerView создаётся класс, унаследованный от RecyclerView.Adapter<VH>, где VH — тип ViewHolder. Необходимо переопределить три метода: 

public class MyAdapter extends RecyclerView.Adapter<MyAdapter.ViewHolder> {

    private List<String> dataList;

    public MyAdapter(List<String> dataList) {
        this.dataList = dataList;
    }

    // 1. Создаёт новый ViewHolder — inflate макета элемента списка.
    //    Вызывается только когда нет переиспользуемых ViewHolder-ов в пуле.
    @NonNull
    @Override
    public ViewHolder onCreateViewHolder(@NonNull ViewGroup parent, int viewType) {
        View view = LayoutInflater.from(parent.getContext())
                .inflate(R.layout.item_list, parent, false);
        return new ViewHolder(view);
    }

    // 2. Привязывает данные к существующему ViewHolder.
    //    Вызывается каждый раз, когда элемент появляется на экране.
    @Override
    public void onBindViewHolder(@NonNull ViewHolder holder, int position) {
        holder.textView.setText(dataList.get(position));
    }

    // 3. Сообщает количество элементов в списке.
    @Override
    public int getItemCount() {
        return dataList.size();
    }

    // ViewHolder — хранит ссылки на View одного элемента списка.
    public static class ViewHolder extends RecyclerView.ViewHolder {
        TextView textView;
        public ViewHolder(@NonNull View itemView) {
            super(itemView);
            textView = itemView.findViewById(R.id.tvItem);
        }
    }
}

2.2 Паттерн ViewHolder

ViewHolder — вложенный класс, кэширующий ссылки на элементы UI одного элемента списка. Без него метод findViewById() вызывался бы при каждой прокрутке, что приводило к обходу всего дерева View и значительному замедлению. ViewHolder устраняет эту проблему: ссылки на View создаются один раз при создании ViewHolder-а, а при повторном использовании — только перезаписываются данные.

2.3 Механизм переиспользования (Recycling)

RecyclerView не создаёт View для каждого элемента данных. Он поддерживает пул (RecycledViewPool) невидимых ViewHolder-ов:

При прокрутке ViewHolder ушедшего за край элемента помещается в пул. Когда появляется новый элемент снизу — берётся ViewHolder из пула и заполняется новыми данными (вызов onBindViewHolder). Таким образом в памяти одновременно находится лишь небольшое количество ViewHolder-ов, независимо от размера списка.

Это позволяет отображать миллионы элементов без переполнения памяти.

2.4 Уведомление об изменениях данных

adapter.notifyDataSetChanged();          // Перерисовать весь список (неэффективно)
adapter.notifyItemInserted(position);    // Добавлен элемент на позицию
adapter.notifyItemRemoved(position);     // Удалён элемент
adapter.notifyItemChanged(position);     // Изменились данные элемента

Для сложных изменений используется DiffUtil — утилита, автоматически вычисляющая минимальное количество операций для перехода от старого списка к новому.

3. Адаптер в ViewPager2

3.1 FragmentStateAdapter

ViewPager2 работает с адаптером типа FragmentStateAdapter. Не случайно: внутри ViewPager2 использует RecyclerView как механизм прокрутки страниц. 

public class TabsPagerAdapter extends FragmentStateAdapter {

    private final int pageCount;

    public TabsPagerAdapter(FragmentActivity activity, int pageCount) {
        super(activity);
        this.pageCount = pageCount;
    }

    // Создаёт Fragment для страницы с указанной позицией.
    @NonNull
    @Override
    public Fragment createFragment(int position) {
        switch (position) {
            case 0:  return new HomeFragment();
            case 1:  return new ProfileFragment();
            case 2:  return new AboutFragment();
            default: return new HomeFragment();
        }
    }

    // Возвращает общее количество страниц.
    @Override
    public int getItemCount() {
        return pageCount;
    }
}

3.2 Управление жизненным циклом фрагментов

FragmentStateAdapter автоматически управляет жизненным циклом фрагментов через FragmentManager:

Состояние фрагмента Что происходит
Становится видимым onCreateView() → onStart() → onResume()
Уходит за offscreenPageLimit onPause() → onStop() → onDestroyView() (состояние сохраняется в Bundle)
Возвращается в область видимости onCreateView() с восстановлением состояния из Bundle

Это принципиально отличает FragmentStateAdapter от устаревшего FragmentPagerAdapter (ViewPager 1), который не уничтожал фрагменты, что приводило к избыточному потреблению памяти.

3.3 Связка TabLayout + ViewPager2

// TabLayoutMediator синхронизирует вкладки и страницы
new TabLayoutMediator(tabLayout, viewPager,
        (tab, position) -> tab.setText(tabTitles[position])
).attach();

TabLayoutMediator — медиатор, который:

  • Автоматически синхронизирует выбранную вкладку TabLayout с текущей страницей ViewPager2
  • Обрабатывает свайп страниц и клики по вкладкам без дополнительного кода
  • Устраняет необходимость вручную слушать OnPageChangeCallback

4. Сравнение адаптеров RecyclerView и ViewPager2

Критерий RecyclerView.Adapter FragmentStateAdapter
Базовый класс RecyclerView.Adapter<VH> RecyclerView.Adapter<VH>
Единица содержимого View (ViewHolder) Fragment
Переиспользование Активное (RecycledViewPool) Ограниченное (offscreenPageLimit)
Управление состоянием Ручное (DiffUtil, notify*) Автоматическое (FragmentManager)
Типичное применение Большие списки данных Страницы/вкладки (3–10 шт.)

5. Вывод

Адаптер в Android — ключевой посредник между данными и UI. В RecyclerView адаптер совместно с паттерном ViewHolder обеспечивает эффективное переиспользование View при работе с большими наборами данных. В ViewPager2 адаптер (FragmentStateAdapter) управляет фрагментами как страницами, интегрируется с FragmentManager для корректного управления жизненным циклом и экономии памяти. Несмотря на разное назначение, оба адаптера базируются на одном внутреннем механизме RecyclerView, что обеспечивает единообразие и предсказуемое поведение системы.
